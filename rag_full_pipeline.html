6. HYPOTHESIS ENGINE (unique feature)
ThreadPoolExecutor  |  LLM  |  Retriever
Step 1 — LLM generates 4 hypotheses as JSON array
Step 2 — Retrieve evidence for each hypothesis in parallel
Step 3 — LLM classifies each chunk: SUPPORTS / CONTRADICTS / IRRELEVANT
Step 4 — Confidence score = (supports − 0.5 × contradicts) / total
Step 5 — LLM generates final ranked verdict

4. Hypothesis Engine — Your Differentiator

No tutorial RAG project has this. Lead with it in interviews.

For a "Why" problem like "Why is bubble sort slower than merge sort?":

1
LLM generates 4 distinct hypotheses as a JSON array
2
Evidence retrieved for each hypothesis in parallel using ThreadPoolExecutor
3
LLM classifies each evidence chunk as SUPPORTS / CONTRADICTS / IRRELEVANT
4
Confidence score computed: (supports − 0.5 × contradicts) / total × 100
5
Hypotheses ranked, verdict generated with sources from your actual PDFs

5. Key Terms Cheat Sheet

RAGRetrieve relevant documents, inject as context into LLM prompt before answering
EmbeddingsText converted to a list of numbers (vectors) that capture meaning — 384 dims here
Cosine SimilarityMeasures angle between two vectors — closer angle = more similar meaning
BM25Keyword-based ranking algorithm — great for exact terms like "bubble sort O(n²)"
Hybrid SearchCombines keyword (BM25) + semantic (vector) search to cover both exact and meaning matches
RRF (Reciprocal Rank Fusion)Algorithm that merges two ranked lists into one by combining their position scores
Cross-encoder RerankingReads query + document together to re-score accuracy (slower but more precise than embeddings)
ChunkingSplitting large documents into small overlapping pieces so LLM context isn't exceeded
Hypothesis EngineGenerates multiple explanations, tests each against documents, ranks by evidence confidence
LangChainPopular framework that wraps all RAG steps — you built each step yourself instead
ChromaDB / PineconeDedicated vector databases — you replaced them with SQLite + NumPy

6. Why Built From Scratch? (Most Important Answer)

"Most people use LangChain which abstracts everything away. I wanted to understand what actually happens under the hood — how embeddings work, how vector similarity search works, why hybrid search is better than just semantic search alone. Building it from scratch means I can explain every line of code in an interview."
